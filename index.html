Upload Meeting Transcripts or Video

You can upload a meeting transcript (.txt or .md) or a video/audio file directly into the LinkUs pipeline to generate a structured leadership communication. The pipeline we built handles all three input types.

📝 Text Transcript
.txt or .md file from Read.ai, Zoom, Teams, or any notetaker. Fastest path.
🎥 Video Recording
.mp4 or .webm file. The pipeline auto-transcribes via speech-to-text, then analyzes.
🎙 Audio Only
.mp3 or .wav file. Same auto-transcription path as video. Works with phone recordings.
How to Upload a Transcript Right Now Place your transcript file in the linkus_pipeline/transcripts/ folder and run:

python3 pipeline.py --transcript transcripts/YOUR_FILE.txt --date "March 18, 2026" --number 2

Connecting Read.ai Directly

Read.ai offers three integration paths. The right one depends on your plan and how automated you want the flow to be.

PATH 1 — EASIEST
Manual Export from Read.ai

After each Tuesday meeting, export the transcript from Read.ai as a text file and drop it into the pipeline folder. Run one command. The communication is generated in under 60 seconds. Works on any Read.ai plan. No API key required.

Step 1: In Read.ai, open the meeting report → Download → Transcript (.txt)
Step 2: Drop the file into linkus_pipeline/transcripts/
Step 3: Run python3 pipeline.py with the file name and meeting number
Result: Full leadership communication generated and saved to /output/
PATH 2 — AUTOMATED
Read.ai Webhook → Auto-Pipeline

Read.ai can automatically push the full meeting data — transcript, summary, action items, participants, and timestamps — to a custom URL the moment the meeting ends. A lightweight server receives it and triggers the pipeline automatically. Requires Read.ai Pro, Enterprise, or Enterprise+ plan.

What Read.ai Sends Automatically After Every Meeting:
✓ Full transcript (speaker + timestamps)
✓ Meeting summary
✓ Action items
✓ Key questions raised
✓ Topics covered
✓ Chapter summaries
✓ All participant names/emails
✓ Meeting report URL

Setup: Read.ai → Account → Integrations → Add Webhook → Enter your server URL. The pipeline server listens, receives the payload, runs the analysis, and emails the communication to the team — all within minutes of the meeting ending.

PATH 3 — ADVANCED
Read.ai REST API / MCP Server

Read.ai launched a REST API and MCP (Model Context Protocol) server in open beta (February 2026). This allows programmatic retrieval of any meeting report, live transcript, or summary on demand. The pipeline can query Read.ai directly — no manual export needed. Available to all Read.ai users. OAuth 2.1 authentication required.

What the API Provides
Meeting reports, live transcripts, summaries, action items — same data as webhooks, pulled on demand.
Current Limitation
OAuth 2.1 tokens expire every 10 minutes. Static API keys are planned but not yet available. Best for on-demand queries rather than fully automated flows.
MCP Integration
Connect Read.ai as an MCP server to Claude, ChatGPT, or the LinkUs pipeline AI — enabling direct meeting data queries from any LLM tool.
No Video Upload via API
Uploading audio/video files for transcription via the API is not yet supported. Use Path 1 or Path 2 for video-based workflows.
Recommended Path for Your Team
Start with Path 1 → Upgrade to Path 2 when ready

Begin this Tuesday: after the meeting, export the transcript from Read.ai and run the pipeline manually. This requires no new setup and works immediately. Once the workflow is proven and the team is comfortable, upgrade to the Webhook path (Path 2) so the communication is generated and distributed automatically — with zero manual steps — the moment the meeting ends.

Next 2–4 Weeks
Validate the output quality over 2–3 meetings, refine the template, confirm the team finds it valuable
Month 2+
Enable Read.ai Webhook → fully automated pipeline → communication sent within minutes of meeting end

The LinkUs Standard meeting has defined roles for each segment. Clarity of ownership ensures the meeting runs on time, stays focused, and produces decisions rather than discussions. The facilitator's job is to protect the agenda — not to fill time.
